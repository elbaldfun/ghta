# Design: 7-add-source-chrome-webstore

## 数据通道（关键不确定性）

Chrome Web Store 无官方公开 API。两条路：

- **A. 第三方数据提供商**（chrome-stats.com 等，付费 API）
  - 优点：结构化、含历史与排行、合规关系清晰（付费授权）、维护成本低。
  - 缺点：费用；依赖第三方可用性与字段。
- **B. 自行抓取**（商店分类/搜索页 + 扩展详情页）
  - 优点：无第三方费用、字段自主。
  - 缺点：无官方榜单需自建"热度"口径（按 users 排序）；受页面改版/反爬/限流影响；ToS 与法律需评估。

> 决策留待实施期：**先评估第三方 API 的成本与授权条款**；可接受则默认走 A（稳），否则受控自抓取走 B。
> 适配器以通道接口抽象，A/B 可切换，核心归一化逻辑共用。

## Decisions

- externalId = 扩展 ID（URL 中 32 位）；primaryMetric = users（desc-better）。
- 无官方排行 → "趋势/排行"由 change 2 基于 users 增长得出，而非现成榜单。
- 抓取（若走 B）：尊重 robots、限速、缓存、失败降级；仅取公开展示数据。

## Risks / Trade-offs

- 合规是首要风险：自抓取需法务/ToS 评估；第三方 API 需授权。文档不预设结论，实施前定夺。
- users 数为区间近似（商店只显示量级），增长计算需容忍粗粒度。
- 通道 B 脆弱：商店改版即可能失效，需监控与快速修复预案。
